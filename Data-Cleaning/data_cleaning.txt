- Data cleaning (also known as data cleansing or data scrubbing) is the process of identifying and correcting errors, 
inconsistencies, and inaccuracies in a dataset.

- In the world of data science, there is a golden rule: "Garbage In, Garbage Out" (GIGO). If you feed a machine learning model or 
an analytical report messy, "dirty" data, the results will be unreliable or flat-out wrong. Data cleaning ensures that your data is 
accurate, complete, and ready for analysis.

- Why Is Data Cleaning Important?
Rarely does data arrive in a perfect state. It often comes from multiple sources with different formats, manual entry errors, or 
technical glitches.4 Cleaning is vital because it:
Improves Accuracy: Removing errors leads to more precise insights.
Increases Efficiency: Clean data is easier for software to process, saving time during the modeling phase.
Enables Better Decisions: Business leaders can trust the reports generated from the data.
Reduces Costs:8 Detecting errors early prevents expensive mistakes later in the production or marketing cycle.


- 5 Common Steps in the Data Cleaning Process:
While the exact steps depend on the dataset, most professionals follow this general workflow:

1. Removing Duplicates and Irrelevant Data: Duplicate entries often happen when data is combined from multiple sources or 
scraped from the web. Removing them prevents "double-counting" and skewed averages. Similarly, if you are analyzing "Customer Age" but 
your dataset includes "Internal Tracking IDs," you should remove the irrelevant columns to keep the dataset lean.

2. Fixing Structural Errors: This involves correcting typos, inconsistent capitalization (e.g., "New York" vs. "new york"), and 
mismatched naming conventions. Example: You might find "N/A" and "Not Applicable" in the same column. Standardizing these into a 
single label ensures they are treated as the same category during analysis.

3. Handling Missing Data: You cannot simply ignore blank cells because many algorithms will fail if they encounter null values. 
You have three main options:
- Drop the observation: Remove the entire row if it’s missing too much critical info.
- Input missing values: Use the mean, median, or mode to "guess" the missing value based on other data.
- Flag it: Leave it blank but mark it so the analysis knows the data was missing.

4. Managing Outliers: Outliers are data points that fall far outside the normal range (e.g., a person's height listed as 10 feet).
- Identify: Use tools like boxplots or scatter plots to find these anomalies.
- Assess: Determine if the outlier is a legitimate rare event or a manual entry error. If it's an error, remove or adjust it.

5. Standardizing and Validating: Finally, ensure that all data follows the same format. This includes:
- Dates: Changing everything to YYYY-MM-DD.
- Units: Converting miles to kilometers or USD to EUR.
- Data Types: Ensuring numbers are stored as "integers" rather than "text" so they can be used in math.
